# 三轮差速小车双模循迹控制系统 —— 技术报告

## 1. 项目概述

本项目基于 ROS1 Noetic + Gazebo 仿真环境，设计并实现了一个三轮差速小车的双模循迹控制系统。系统融合**摄像头视觉处理**与**虚拟光电传感器阵列**两种感知方式，实现小车在闭合赛道上的稳定循迹。

### 1.1 项目来源

基于许德新老师的两个开源项目：

- `smart_car_follower`：五路光电传感巡线仿真
- `smart_car_no_mask`：摄像头视觉巡线仿真（本项目基础）

### 1.2 核心特性

| 特性 | 说明 |
|------|------|
| 双模感知 | 摄像头图像处理 + 5路光电传感器阵列 |
| 融合方式 | 卡尔曼滤波 + 自适应观测噪声 |
| PID 控制 | 可调 PID 参数 + 弯道自动减速 |
| 丢线恢复 | 前弧搜索 → 倒车弧 → 交替扫描 状态机 |
| Web 调参 | 浏览器实时调参面板（端口 8091） |
| RViz 可视化 | 赛道、传感器状态、误差箭头实时显示 |

## 2. 系统架构

### 2.1 硬件模型

三轮差速小车 URDF 模型包含：

- **底盘**：340mm × 220mm、蓝色车体
- **驱动轮**：左右两个独立驱动轮（半径 60mm），差速转向
- **万向轮**：前部支撑轮
- **摄像头**：前置下倾，640×480@30fps，80° FOV
- **传感器条**：前端搭载 5 路光电传感器阵列

### 2.2 软件架构

```
Gazebo 仿真环境
    │

    │
    ├── 虚拟光电传感器
    │   └── line_sensor_node
    │       ├── 从 Gazebo 获取小车位姿
    │       ├── 计算 5 路传感器世界坐标
    │       ├── 采样赛道灰度值模拟 ADC
    │       ├── 二值化阈值判断
    │       └── 偏差计算 → /line_follower/sensor_error

    ├── 传感器融合节点

```

## 3. 摄像头视觉处理

### 3.1 图像预处理

```python
# ROI 裁剪：取图像下半部 55%-95% 区域
roi = frame[y1:y2, x1:x2]
# 灰度转换 + 高斯模糊去噪
gray = cv2.cvtColor(roi, cv2.COLOR_BGR2GRAY)
blur = cv2.GaussianBlur(gray, (5, 5), 0)
```

### 3.2 阈值分割

支持三种阈值模式：

| 模式 | 方法 | 适用场景 |
|------|------|---------|
| Otsu（默认） | 大津法自动计算最佳阈值 | 光照均匀的赛道 |
| 固定阈值 | 手动设定 black_threshold | 光照稳定的环境 |
| 自适应阈值 | 局部邻域阈值 | 光照不均的赛道 |

### 3.3 形态学处理

经过阈值分割的二值图像存在噪点，通过形态学运算消除：

```python
kernel = cv2.getStructuringElement(cv2.MORPH_ELLIPSE, (5, 5))
mask = cv2.morphologyEx(mask, cv2.MORPH_OPEN, kernel)   # 去噪点
mask = cv2.morphologyEx(mask, cv2.MORPH_CLOSE, kernel)  # 填孔洞
```

### 3.4 轮廓筛选与加权质心

找到所有轮廓后，按面积筛选出代表黑线的主轮廓。在选定轮廓上计算加权质心：

```python
# 加权质心：底部（近车端）权重更高
weights = 0.25 + 0.75 * (ys / h)  # 底部权重最大为顶部的 4 倍
cx = average(xs, weights=weights)
```

加权策略使系统能提前感知弯道——当车头刚偏离切线时，近端的线已偏移，产生非零误差。

### 3.5 误差计算

```python
error = (cx - roi_width/2) / (roi_width/2) * 2.0
error = clamp(error, -2.0, 2.0)
```

误差范围归一化到 [-2.0, +2.0]，负值表示黑线偏左，正值表示偏右。

## 4. 虚拟光电传感器

### 4.1 传感器布局

5 路光电传感器以 9mm 间距等距排列，位于车前方 180mm 处：

```
       ↑ 小车前进方向
       |
   ┌───┼───┐
   ◎   ◎   ◎   ◎   ◎
  [0] [1] [2] [3] [4]
  左2 左1 中心 右1 右2
  权重：-2  -1   0   +1   +2
```

### 4.2 ADC 模拟

每个传感器通过采样赛道几何位置获取 ADC 值：

- 白底（高反射率）：ADC ≈ 3500
- 黑线（低反射率）：ADC ≈ 800
- 添加 ±30 随机噪声模拟实际抖动

### 4.3 偏差计算

支持三种算法：

| 算法 | 说明 |
|------|------|
| 二值化重心法（默认） | ADC 阈值比较 → 重心加权 |
| 模拟值加权法 | ADC 灰度直接加权 |
| 抛物线插值法 | 最小值抛物线拟合，亚像素精度 |

## 5. 传感器融合

### 5.1 融合策略

融合算法历经两版迭代：

**V1：固定加权平均（初始方案）**

```python
error = 0.6 × camera_error + 0.4 × sensor_error
```

问题：权重固定，无法根据传感器质量动态调整。任一传感器受干扰时，融合输出被污染。

**V2：卡尔曼滤波融合（当前方案）**

核心思想：将线偏差建模为状态变量，两个传感器为独立观测源，通过卡尔曼增益动态融合。

#### 状态模型

```
状态向量：x = [error, error_rate]^T
              ─ 线偏差     ─ 偏差变化率

预测模型：x_k = F @ x_{k-1} + w,   w ~ N(0, Q)

F = [[1, dt],      Q = [[σ_p·dt, 0   ],
     [0,  1]]            [0,      σ_p·2]]

观测模型：z_camera = H @ x + v_c,   v_c ~ N(0, R_camera)
          z_sensor = H @ x + v_s,   v_s ~ N(0, R_sensor)

H = [[1, 0]]    （直接测量 error 分量）
```

#### 自适应观测噪声

核心创新：观测噪声协方差 R 由传感器置信度动态调节。

```python
# 滚动方差估计（窗口 10 帧）
camera_var = rolling_variance(camera_errors, window=10)
sensor_var = rolling_variance(sensor_errors, window=10)

# 自适应 R：方差越大 → R 越大 → 卡尔曼增益越小 → 权重越低
R_camera = R_base × (1 + camera_var / R_base)
R_sensor = R_base × (1 + sensor_var / R_base)

# 离线传感器 R 极大（≈不信任）
if not camera_online: R_camera = 10.0
if not sensor_online:  R_sensor = 10.0
```

| 场景 | camera_var | sensor_var | 效果 |
|------|-----------|-----------|------|
| 正常循线 | 低 | 低 | 均衡融合 |
| 摄像头抖动 | 高 | 低 | 传感器占主导 |
| 传感器噪声 | 低 | 高 | 摄像头占主导 |
| 任一离线 | — | — | 完全信任另一路 |

#### 卡尔曼更新流程

每步执行：
1. **预测**：x_pred = F·x；P_pred = F·P·Fᵀ + Q
2. **计算 R**：根据滚动方差 + 在线状态
3. **更新 camera**：K = P·Hᵀ / (H·P·Hᵀ + R_cam)；x += K·(z_cam - H·x)
4. **更新 sensor**：同上，用 R_sen
5. **发布**：fused_error = x[0]，clamped to [-2, 2]

#### 卡尔曼优势

| 特性 | 固定加权 | 卡尔曼滤波 |
|------|---------|-----------|
| 噪声抑制 | ❌ 无 | ✅ 状态估计天然滤波 |
| 自适应权重 | ❌ 固定 | ✅ 根据方差自动调节 |
| 时序预测 | ❌ 无 | ✅ 含运动模型预测 |
| 单传感器失效 | ⚠️ 权重不变 | ✅ 自动降权至 0 |

### 5.2 融合优势

| 场景 | 摄像头 | 光电传感器 | 融合结果 |
|------|--------|-----------|---------|
| 正常循线 | ✅ 有效 | ✅ 有效 | 加权平均 |
| 摄像头被遮挡 | ❌ online=False | ✅ 有效 | 仅用传感器 |
| 传感器噪声 | ✅ 有效 | ⚠️ 权重降低 | 以摄像头为主 |
| 光线变化 | ⚠️ 阈值偏移 | ✅ 不受影响 | 以传感器为主 |

## 6. PID 控制器

### 6.1 控制律

```
steer = Kp × error + Ki × ∫error·dt + Kd × d(error)/dt
```

| 参数 | 默认值 | 作用 |
|------|--------|------|
| Kp | 32.0 | 比例增益，决定响应速度 |
| Ki | 0.5 | 积分增益，消除稳态误差 |
| Kd | 1.0 | 微分增益，抑制振荡 |

### 6.2 差速驱动

```
左轮速度 = base + steer
右轮速度 = base - steer
```

### 6.3 弯道减速

```python
base = clamp(base_speed - |steer| × curve_factor, min_speed, base_speed)
```

弯道转向量 (steer) 大时自动降低基础速度，直道恢复全速。

### 6.4 丢线恢复状态机

| 阶段 | 动作 | 持续时间 |
|------|------|---------|
| 前弧搜索 | 前进 + 转向 | 0.75s |
| 倒车弧 | 后退 + 反向转向 | 0.55s |
| 交替扫描 | 左右交替 | 1.2s/周期 |
| 安全停止 | 停车 | 超时触发 |

| 赛道半径 | ~2-2.5m |

| 参数 | 值 |

| 驱动方式 | 差速轮式 |
| 车轮半径 | 60mm |
| 轮距 | 300mm |
| 最大线速度 | 0.45 m/s |
| 最大角速度 | 2.8 rad/s |

```bash
docker run -d --rm \
  --name smart_car \
  --network host \
  --gpus all \
```
